1 Project Overview

SmartHire is a full-stack AI-powered job portal that connects job seekers with recruiters through intelligent skill-based job matching, built on the MERN stack (MongoDB, Express, React, Node.js).

1.1 Abstract

SmartHire addresses the problem of irrelevant job recommendations by implementing a custom skill-matching algorithm that computes a compatibility score between a seeker's skills and a job's requirements. Unlike keyword-based search systems, this approach quantifies match quality as a percentage, enabling ranked and filtered recommendations that genuinely fit the candidate's profile.

The platform supports two user roles вҖ” Job Seeker and Recruiter вҖ” each with dedicated dashboards, profile management, and workflow tools. The application uses JWT-based authentication, bcrypt password hashing, and role-based route protection throughout.

1.2 Objectives

  • Build a complete full-stack job portal using the MERN stack
  • Implement a skill-based recommendation algorithm (match score вүҘ 40%)
  • Provide dual-role authentication (seeker and recruiter)
  • Enable real-time application tracking with status management
  • Create a responsive, modern dark-mode UI using Tailwind CSS
  • Support resume upload, job bookmarking, and advanced search/filter

6 AI Recommendation Engine

рҹ“җ Match Score Formula

Match Score = (Number of Matching Skills Г· Total Required Skills) Г— 100

A job is recommended if its match score is вүҘ 40%. Results are sorted in descending order by score.

6.1 Algorithm Implementation

// server/utils/matchScore.js

function matchScore(seekerSkills, jobSkills) {
  if (!jobSkills || jobSkills.length === 0) return 0;
  if (!seekerSkills || seekerSkills.length === 0) return 0;

  // Case-insensitive comparison
  const normalizedSeeker = seekerSkills.map(s => s.toLowerCase().trim());
  const matches = jobSkills.filter(skill =>
    normalizedSeeker.includes(skill.toLowerCase().trim())
  );

  return Math.round((matches.length / jobSkills.length) * 100);
}

function getRecommendedJobs(seekerSkills, jobs, threshold = 40) {
  return jobs
    .map(job => ({
      ...job.toObject(),
      matchScore: matchScore(seekerSkills, job.requiredSkills)
    }))
    .filter(job => job.matchScore >= threshold)  // 40% minimum
    .sort((a, b) => b.matchScore - a.matchScore); // descending
}

8.2 JWT Token Lifecycle

// Token generation (server)
const token = jwt.sign(
  { id: user._id },
  process.env.JWT_SECRET,
  { expiresIn: '7d' }   // 7-day expiry
);

// Token storage (client)
localStorage.setItem('smarthire_token', token);

// Auto-attach on every request (Axios interceptor)
api.interceptors.request.use(config => {
  const token = localStorage.getItem('smarthire_token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Auto-logout on 401 (Axios interceptor)
api.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      localStorage.clear();
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

11 Conclusion

SmartHire successfully demonstrates a production-grade, full-stack MERN application with intelligent job matching, dual-role authentication, and a polished user experience.

11.1 What Was Achieved

  • Built a complete 3-tier MERN application from scratch with clean separation of concerns
  • Implemented a custom skill-based recommendation engine with configurable match threshold
  • Created 6 MongoDB collections with proper indexing and relationships
  • Delivered 16 REST API endpoints with JWT auth and role-based access control
  • Built 8 fully functional React pages with responsive design and micro-animations
  • Used modern tooling: Vite for fast HMR, Tailwind for rapid UI development, Axios interceptors for auth

11.2 Challenges & Solutions

Challenge Solution
Preventing duplicate applications Compound unique index on {jobId, seekerId} in Application schema
Route ordering conflict (recommended vs :id) Placed named routes before parameterized routes in Express
Case-insensitive skill matching Normalized both arrays to lowercase before comparison
JWT persisting across page refreshes Stored token in localStorage and verified on AuthContext mount
Multer file validation MIME type check (application/pdf) + 5MB size limit
